
# Resumo  Git e GitHub


Resumo sobre alguns comandos em markdown para usar no curso de versionamento de código com Git e posteriormente salvar no GitHub.

Testando o uso de parágrafo. Para que fique mais organizado o texto.

Para um texto mais organizado é preciso seguir algumas convenções markdown.

## Redução de linha

Para uma quebra de linha ou uma linha nova, termine essa linha com ou mais espaçose digite return.

Caso contrário pode ser usado uma tag html, caso o seu editor de texto sea compatível. Use o comando ao final da sua linha.<br>

## Negrito ou Itálico

Para um textoem negrito usa dois asteriscos no início da palavra ou frase, por exemplo: Este texto está em **negrito**.<br>

Para usar um texto em itálico use apenas um asterisco antes e depois da palavra ou frase. Exemplo, está pavra está em *itálico*.

## Links e endereços

Os aplicativos markdown não existe um cosenso sobre o uso de espaços no meio da url porém aqui vai um exemplo, de como usar links. Para saber mais sobre a documentação markdown segue o [link](https://readme.so/pt/editor) para a documentação.

## Usando imagens em seu texto
